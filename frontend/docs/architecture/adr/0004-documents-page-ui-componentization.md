# ADR 0004: Декомпозиция UI страницы Documents

- **Статус:** Proposed
- **Дата:** 2026-02-25
- **Автор:** Iakov
- **Область (Scope):** Frontend (Documents)

## Контекст (Context)

Страница Documents разрослась и содержит несколько крупных UI-блоков (upload flow, processing status, results, error). Даже после выноса API-логики в hooks, UI остается монолитным. Это ухудшает читаемость, усложняет тестирование и замедляет изменения в отдельных секциях.

Нужно разделить UI на компоненты без изменения поведения.

## Варианты решения (Considered Options)

1. **Вариант А:** Оставить весь UI в page.tsx.
2. **Вариант Б:** Вынести UI-блоки в отдельные компоненты.

## Решение (Decision Outcome)

**Выбран вариант Б с уточнением.** Не все UI-блоки требуют отдельного компонента. Критерий: компонент имеет смысл, если он содержит условную логику или переиспользуется.

- **UploadSection** (заголовок + Dropzone) и **StrategySection** (заголовок + Selector) — это простые обёртки без логики. Оставляем их в `page.tsx` чтобы не добавлять лишней абстракции.
- **ActionButtons** (содержит условное отображение Reset) и **JobResult** (объединяет 3 состояния: processing/completed/failed) — выносим в отдельные компоненты, так как они содержат логику и могли бы переиспользоваться.

### Обоснование (Pros)

- Улучшается читаемость за счёт разделения логики, не за счёт лишней абстракции.
- Упрощается тестирование и переиспользование компонентов с логикой.
- Снижается когнитивная нагрузка при изменениях в картах результатов (JobResult) и кнопках действий (ActionButtons).
- Избегаем избыточной абстракции для простых обёрток (h2 + компонент).

### Недостатки (Cons)

- Больше файлов и импортов.
- Появляется прокидывание пропсов между компонентами.

## Последствия (Consequences)

- Создать 2 компонента UI для страницы Documents: `action-buttons.tsx` и `job-result.tsx`.
- Обновить `page.tsx`, чтобы он объединял основную разметку (upload, strategy) с компонентами.
- Поведение страницы остается прежним.
- Избежали лишней абстракции, сохранили читаемость кода.

## Структура файлов

```
frontend/
└── app/
	└── dashboard/
		└── documents/
			├── page.tsx
			└── components/
				 ├── action-buttons.tsx
				 └── job-result.tsx
## Состав компонентов
```

- **action-buttons.tsx**: основная кнопка (Instant Analyze / Create Analysis Job) и кнопка Reset (условно отображается).
- **job-result.tsx**: отрисовка состояния processing/completed/failed (карточка "Processing Job", результаты или ошибка + кнопка "Try Again").

**В page.tsx остаются:**

- Dropzone + заголовок "1. Upload Document"
- Strategy Selector + заголовок "2. Choose Analysis Method"

Эти блоки слишком просты для отдельного компонента (h2 + компонент), поэтому оставляем их на месте.
