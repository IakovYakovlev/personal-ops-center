# ADR 0001: Ограничение количества чатов на пользователя

- **Статус:** Accepted
- **Дата:** 2026-03-22
- **Автор:** Jakov
- **Область (Scope):** neural-assistant

## Контекст (Context)

Пользователь может создавать неограниченное количество чатов, что приводит к росту нагрузки на базу данных и усложняет управление историей. Необходимо ограничить число активных чатов для каждого пользователя, чтобы обеспечить предсказуемое использование ресурсов и упростить UX.

## Варианты решения (Considered Options)

1. **Вариант А:** Не ограничивать количество чатов (оставить как есть).
2. **Вариант Б:** Ввести лимит на количество чатов на пользователя и автоматически удалять самый старый чат при создании нового сверх лимита.

## Решение (Decision Outcome)

Выбран Вариант Б. В системе устанавливается максимальное число чатов на пользователя (например, 3). Если пользователь создает новый чат, когда лимит уже достигнут, система автоматически удаляет самый старый чат и сохраняет новый.

### Обоснование (Pros)

- Контролируемое потребление ресурсов.
- Упрощение пользовательского интерфейса (нет бесконечного списка чатов).
- Предсказуемое поведение для пользователя.

### Недостатки (Cons)

- Пользователь может потерять старую историю чата без предупреждения.
- Необходимость реализовать логику удаления и уведомления пользователя.

## Последствия (Consequences)

- При создании нового чата система проверяет лимит и удаляет самый старый чат, если лимит превышен.
- Пользователь всегда видит только последние N чатов (например, 3).
- Требуется реализовать удаление чата и обновление UI на фронтенде.
