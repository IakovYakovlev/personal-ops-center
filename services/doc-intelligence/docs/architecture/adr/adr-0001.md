# ADR 0001: Ограничение количества документов пользователя

- **Статус:** Proposed
- **Дата:** 2026-03-24
- **Автор:** Jakov
- **Область (Scope):** doc-intelligence

## Контекст (Context)

В демонстрационном проекте пользователи могут загружать документы для анализа и эмбединга. Без ограничений возможно переполнение базы данных, рост затрат на хранение и снижение производительности. Необходимо ограничить количество документов, хранимых на пользователя, чтобы система оставалась стабильной и предсказуемой.

## Варианты решения (Considered Options)

1. **Вариант А:** Не ограничивать количество документов.
2. **Вариант Б:** Ввести лимит (например, 5 документов) и удалять самый старый при загрузке нового.

## Решение (Decision Outcome)

Выбран Вариант Б. Для каждого пользователя хранится не более 5 документов. При загрузке нового документа, если лимит превышен, автоматически удаляется самый старый документ.

### Обоснование (Pros)

- Контролируемый объем хранения.
- Предсказуемые расходы на инфраструктуру.
- Простота реализации.

### Недостатки (Cons)

- Пользователь может потерять старые документы без предупреждения.
- Необходимость информировать пользователя о лимите.

## Последствия (Consequences)

- Требуется реализовать логику удаления старых документов при превышении лимита.
- Желательно добавить уведомление пользователю о лимите и удалении.
- Возможна доработка UI для отображения текущего количества документов.
